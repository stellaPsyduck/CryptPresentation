\begin{frame}{Homomorphic Encryption} % Stella
	Ability to apply transformations to encrypted data, without the need to decrypt. \\ 
	%**Talk about application to stats and ai since the data can be hidden but the model can still be performed**
	\vspace{0.5cm}
	$\varepsilon$ is homomorphic for circuits in $C_{\varepsilon}$ if $\varepsilon$ is correct for $C_{\varepsilon}$ and
	$\text{Decrypt}_{\varepsilon}$ can be expressed as a circuit $D_{\varepsilon}$ of size poly($\lambda$). \\
			% So homomorphic equations have transformations (which are called circuits) on encrypted data but there's definitions on how much one can handle.
		% Basic one is like RSA, the epslion is the encryption sscheme and this is essentially saying the scheme is properly homorphic if the encrypted and transformed 
		% data is outputting the righ thing with some security level lambda. Important to note, this definition from our paper asks the Decrpytion to be represented as a circuit as well
	\vspace{0.5cm}
	
	\begin{itemize}
	\item \textbf{Encrpytion Scheme ($\varepsilon$)} KeyGen, Encrypt, Decrypt, Evaluate % Sounds familiar but not quite the same
	\item RSA is not \textit{fully} HE, only \textit{levelled} HE
	\end{itemize} 

\end{frame}

\begin{frame}{Formal Definition} % Stella
	
	\underline{Circuit:} Fancy way to show equations through a digraph. Each node is a binary operation for addition or multiplication. \\

	\begin{itemize}
	\item \textbf{Levelled Homomorphic Encryption: }$\varepsilon$ is homorphic for circuits up to a certain depth. Defined into families of $\varepsilon^{\{d\}}$ where d is the depth. \\ % Explain depth w picture
	\item \textbf{Fully Homomorphic Encryption: }$\varepsilon$ is homomorphic for all circuits
